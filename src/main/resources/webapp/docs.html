Arquitectura del Sistema LoopFlow 2.0

Última revisión técnica: Mayo de 2026 · Especificaciones del núcleo SPA de alto rendimiento.

LoopFlow 2.0 está estructurado como una Single Page Application (SPA) monolítica del lado del cliente, desacoplada de dependencias pesadas de compilación. Su diseño persigue la reactividad nativa mediante Vanilla JavaScript y el aislamiento modular de la capa de datos.

1. Infraestructura y Ciclo de Ejecución

A diferencia de las arquitecturas web tradicionales atadas a empaquetadores como Webpack, Vite o TurboPack, el núcleo de LoopFlow procesa los scripts en tiempo de ejecución de manera nativa mediante módulos ES6. Esto elimina la fase de compilación (*build-less framework deployment*), agilizando el flujo de trabajo local.

Protocolos de Red Obligatorios

Debido a las estrictas políticas de seguridad (*CORS* y resolución de rutas dinámicas de módulos), la aplicación no puede ejecutarse abriendo el archivo directamente en el navegador mediante el protocolo de archivos file://. Es mandatorio montar un socket de red local bajo HTTP/HTTPS:

  • Entornos de Desarrollo: Utilice instancias HTTP ligeras como la extensión Live Server en VS Code o entornos CLI de Node.js mediante el comando npx serve . en la raíz del proyecto.
  • Soporte de API: Las conexiones asíncronas asumen la disponibilidad persistente de los archivos JS encargados de inyectar los controladores globales en el espacio de nombres de la ventana de ejecución (window).

2. Gestión de Estado Global Asíncrono

El flujo de datos del sistema está centralizado en el objeto global inmutable window.LoopFlowAPI. Esta capa intercepta las peticiones de las vistas, actúa como abstracción intermedia de persistencia y garantiza que los componentes visuales permanezcan puros y libres de lógica de negocio directa.

Nota de Diseño de Ingeniería

El desacoplamiento de LoopFlowAPI permite realizar una migración transparente desde el almacenamiento en caché local (*LocalStorage / IndexedDB*) hacia pasarelas distribuidas o servicios REST externos remotos utilizando la Fetch API nativa, sin necesidad de alterar una sola línea de código CSS o HTML en las interfaces del Dashboard, Hábitos o Tablero Scrum.

Flujo Reactivo del Dashboard

El módulo del panel central procesa eventos mediante un bucle de actualización síncrona/asíncrona disparado por el DOM. Al inicializarse la vista, el método ejecutor loadDashboard() realiza un mapeo completo de la base de datos distribuida:

  1. Invoca la promesa unificada window.LoopFlowAPI.dashboard.getSummary().
  2. Resuelve la inyección dinámica de datos numéricos en los contadores clave mediante manipulación selectiva del DOM utilizando selectores únicos de ID.
  3. Calcula en tiempo de ejecución la Tasa de Flujo operacional empleando la fórmula de rendimiento de hábitos diarios:
Tasa de Flujo = (habitsCompletedToday / totalHabits) * 100

3. Motor de Sanitización y Capa Antifraude

Dado que el sistema permite la inserción activa de cadenas de texto proporcionadas por el usuario al crear nuevos hábitos o definir subtareas en el tablero Scrum, el núcleo implementa medidas de defensa perimetral contra inyecciones de código malicioso de tipo Cross-Site Scripting (XSS).

Toda inserción dinámica mapeada mediante plantillas literales de JavaScript (Template Literals) filtra obligatoriamente las entradas de texto a través del método seguro de conversión de entidades HTML window.escapeHtml(). Esto transforma caracteres críticos como <, >, y " en secuencias codificadas inertes que el navegador procesará estrictamente como texto plano, inhabilitando la ejecución de scripts arbitrarios en memoria.

4. Catálogo y Especificación de Métodos de la API

A continuación se detalla la firma técnica de los servicios públicos disponibles en el namespace global de desarrollo del ecosistema LoopFlow:

ASYNC GET LoopFlowAPI.dashboard.getSummary()

Resuelve un objeto agregado conteniendo matrices completas sobre los hábitos activos globales, contadores de actividades programadas y el backlog filtrado de tareas en curso del tablero Kanban.

ASYNC POST LoopFlowAPI.habits.complete(habitId, status, logMessage)

Modifica de forma atómica la marca de tiempo de cumplimiento de un hábito específico. Recibe el identificador numérico entero, un booleano de estado y una cadena descriptiva para auditoría de rendimiento.

ASYNC GET LoopFlowAPI.categories.getAll()

Retorna la colección completa de clústeres organizacionales mapeados en el sistema junto con sus valores cromáticos hexadecimales asociados de representación gráfica.

5. Directrices del Sistema de Diseño Visual

La estética visual está inspirada en las guías normativas de desarrollo limpio de Google. Las interfaces se rigen por tres principios estandarizados:

  • Tratamiento Iconográfico Homogéneo: Queda terminantemente prohibido el uso de colecciones tipográficas tipificadas como Emojis nativos de color debido a su disparidad estética entre sistemas operativos (iOS, Android, Windows). Se exige la implementación exclusiva de símbolos Unicode Geométricos puros, calibrados visualmente mediante opacidades suaves y escalas tipográficas relativas.
  • Persistencia de Modos de Iluminación: El motor de renderizado gestiona los perfiles mediante la inserción de la clase .dark al nodo raíz del DOM. Esto sincroniza de forma inmediata todas las variantes estéticas declaradas mediante clases de utilidad atómicas provistas por Tailwind CSS.
